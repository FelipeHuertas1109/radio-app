# Guía de Contribución - HQS Radio

Gracias por tu interés en contribuir a HQS Radio.

## Estructura del Código

- `App.js` - Punto de entrada principal
- `components/RadioPlayer.js` - Lógica del reproductor
- `components/AdminPanel.js` - Panel de administración

## Convenciones de Código

- Usar componentes funcionales con hooks
- Comentar funciones complejas
- Seguir el estilo de código existente
- Usar nombres descriptivos para variables y funciones

## Proceso de Desarrollo

1. Crear una rama para tu feature
2. Hacer cambios y probar localmente
3. Asegurarse de que no haya errores de linting
4. Crear un pull request con descripción clara

## Pruebas

Antes de enviar cambios:
- Probar en iOS (si es posible)
- Probar en Android (si es posible)
- Verificar que no haya errores de consola
- Probar la funcionalidad principal

## Reportar Bugs

Al reportar bugs, incluye:
- Descripción del problema
- Pasos para reproducir
- Comportamiento esperado vs actual
- Información del dispositivo/OS
- Logs de error (si aplica)
